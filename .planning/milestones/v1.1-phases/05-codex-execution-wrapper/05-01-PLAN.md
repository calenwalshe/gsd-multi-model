<task type="auto" tdd="true" executor="claude" confidence="high">
  <name>Task 1: Create bin/codex-task.sh — Codex execution wrapper</name>
  <files>bin/codex-task.sh</files>
  <behavior>
    - --plan 04-01-PLAN.md --task 1: extracts task 1 XML, parses executor/confidence, invokes Codex in worktree, returns JSON report
    - --plan 04-01-PLAN.md --task 1 --dry-run: prints parsed task, prompt, and Codex command without executing
    - --plan 04-01-PLAN.md --task 1 --timeout 60: kills Codex after 60 seconds, exits 3
    - --plan 04-01-PLAN.md --task 1 --force: allows running even if task executor != codex
    - No --plan or --task argument: exits 4 with usage message
    - Plan file does not exist: exits 2 with parse error message
    - Task number not found in plan: exits 2 with parse error message
    - codex not in PATH: exits 4 with install instructions
    - Task has executor="claude" without --force: warns and exits 4
    - Task has confidence="low": warns and exits 4 with escalation message
    - Codex exits non-zero: exits 1, includes Codex stderr in report
    - Codex times out: exits 3, includes partial output if any
  </behavior>
  <action>
    Create `bin/codex-task.sh` following the same shell conventions as `bin/worktree-create.sh` (set -euo pipefail, ANSI colors with TTY detection, ok/warn/err helpers on stderr).

    **CLI interface:**
    - `bin/codex-task.sh --plan PATH --task N` — required arguments
    - `bin/codex-task.sh --plan PATH --task N --dry-run` — print without executing
    - `bin/codex-task.sh --plan PATH --task N --timeout SECONDS` — override default 300s timeout
    - `bin/codex-task.sh --plan PATH --task N --force` — skip executor validation
    - `bin/codex-task.sh --plan PATH --task N --json` — (always JSON to stdout, but this flag exists for consistency with other bin/ scripts; JSON is the default output mode)

    **Structure the script in these sections:**

    **Section 1 — Argument parsing and pre-flight checks:**
    1. Parse CLI arguments: --plan, --task, --dry-run, --timeout (default 300), --force
    2. Validate --plan and --task are provided (exit 4 if missing)
    3. Validate plan file exists (exit 2 if not)
    4. Check `codex` in PATH — if missing and not --dry-run, exit 4 with: "Codex CLI not found. Install with: npm install -g @openai/codex"
    5. Check git repo (exit 4 if not in git repo)

    **Section 2 — XML task extraction (CODEX-02):**
    Parse the PLAN.md to extract the Nth task block using shell tools (grep/sed/awk — no external parser per user decision).

    Algorithm:
    1. Read the plan file
    2. Extract the Nth task XML block (opening tag through closing tag). Use awk to count task blocks and extract the one matching the requested number.
    3. From the task block, extract these attributes and child elements:
       - `executor` attribute from opening tag (grep -oP 'executor="[^"]*"')
       - `confidence` attribute from opening tag
       - name element content (task name)
       - files element content (files list)
       - action element content (full action text — this becomes the Codex prompt)
       - done element content (acceptance criteria)
    4. From plan frontmatter, extract `files_modified` list (the YAML array)
    5. Derive task_id from plan filename + task number: e.g., "04-01-T1"

    **Executor validation:**
    - If executor != "codex" and --force not set: warn "Task executor is '$executor', not 'codex'. Use --force to override." and exit 4
    - If executor != "codex" and --force set: warn but proceed

    **Confidence routing (per user decision):**
    - confidence="high" → CODEX_MODE="--full-auto"
    - confidence="medium" → CODEX_MODE="" (default approval mode)
    - confidence="low" → warn "Low confidence task — escalate to Claude instead" and exit 4

    **Section 3 — Context injection and prompt building (CODEX-01):**
    Build the Codex prompt by concatenating:
    1. "You are working on a planned task. Follow the instructions exactly."
    2. Task name and task_id
    3. The full `<action>` content from the XML block
    4. "Acceptance criteria: " + `<done>` content
    5. "Files to modify: " + files_modified list
    6. If CLAUDE.md exists in repo root, append: "Project instructions (follow these conventions):\n" + contents of CLAUDE.md
    7. If AGENTS.md exists in repo root, append: "Build & architecture conventions:\n" + contents of AGENTS.md

    Store the prompt in a temp file (mktemp) for passing to Codex.

    **Section 4 — Dry-run output:**
    If --dry-run flag is set:
    - Print to stderr: parsed task name, executor, confidence, files_modified, task_id
    - Print to stderr: the constructed prompt (truncated to first 500 chars)
    - Print to stderr: the Codex command that would be run
    - Print JSON to stdout: `{"dry_run": true, "task_id": "...", "executor": "...", "confidence": "...", "codex_command": "codex ... --full-auto ..."}`
    - Exit 0

    **Section 5 — Worktree creation:**
    1. Run `bin/worktree-create.sh --task "$PLAN_FILE" --json`
    2. Parse JSON output to get `branch` and `path`
    3. If worktree creation fails, exit 4 with the error

    **Section 6 — Codex invocation:**
    1. `cd` into the worktree path
    2. Record start time (`date +%s`)
    3. Run Codex with timeout: `timeout $TIMEOUT codex $CODEX_MODE --quiet -p "$(cat $PROMPT_FILE)" 2>"$STDERR_FILE" | tee "$STDOUT_FILE"`
       - Capture exit code
       - Use temp files for stdout and stderr capture
    4. Record end time, calculate duration
    5. Handle exit codes:
       - 0: success, proceed to commit
       - 124: timeout, set exit_reason="timeout"
       - Other: Codex failure, set exit_reason="codex_error"

    **Section 7 — Auto-commit and result capture (CODEX-03):**
    On Codex success (exit 0):
    1. `cd` into worktree path
    2. Stage all changes: `git add -A`
    3. Check if there are changes to commit (`git diff --cached --quiet`)
    4. If changes exist, commit with message: `feat($TASK_ID): $TASK_NAME`
    5. Capture commit hash: `git rev-parse --short HEAD`
    6. Capture changed files: `git diff --name-only HEAD~1` (as JSON array)
    7. Capture diff stats: insertions/deletions per file from `git diff --stat HEAD~1`

    **Section 8 — Worktree cleanup:**
    1. `cd` back to original repo root
    2. Determine cleanup behavior (Claude's discretion: keep worktree on failure for debugging):
       - On success: run `bin/worktree-cleanup.sh $BRANCH --json` to merge back
       - On failure/timeout: run `bin/worktree-cleanup.sh --no-merge --force $BRANCH` to discard
       - Capture cleanup result
    3. Clean up temp files (prompt, stdout, stderr)

    **Section 9 — Structured JSON output (CODEX-03):**
    Print JSON to stdout with all fields per user decision:
    ```json
    {
      "exit_code": 0,
      "changed_files": ["file1.ts", "file2.ts"],
      "commit_hash": "abc1234",
      "task_id": "04-01-T1",
      "duration_seconds": 45,
      "plan": "04-01-PLAN.md",
      "executor": "codex",
      "confidence": "high",
      "diff_summary": {"file1.ts": {"+": 10, "-": 3}, "file2.ts": {"+": 5, "-": 0}},
      "merge_commit": "def5678",
      "codex_stdout": "...",
      "codex_stderr": "..."
    }
    ```

    On failure, still output JSON but with non-zero exit_code, empty changed_files/commit_hash, and include codex_stderr.

    Print human-readable summary to stderr:
    ```
    ═══════════════════════════════════════════════════════
     GSD Codex Task Complete
    ═══════════════════════════════════════════════════════
      Task:        04-01-T1: Task Name
      Status:      success | failed | timeout
      Duration:    45s
      Files:       2 changed (+15, -3)
      Commit:      abc1234 (merged as def5678)
    ═══════════════════════════════════════════════════════
    ```

    **Exit codes (per user decisions):**
    - 0: success
    - 1: Codex failure (non-zero exit from Codex)
    - 2: parse error (plan not found, task not found, XML parse failure)
    - 3: timeout
    - 4: pre-flight failure (missing codex, missing args, executor mismatch, low confidence)

    Make the script executable (`chmod +x`).
