# Requirements: gsd-multi-model

**Defined:** 2026-03-05
**Core Value:** Structured dual-tool workflow that splits work by tool strengths automatically

## v1.2 Requirements

### Compatibility

- [x] **COMPAT-01**: `gsd-compat.json` exists in repo root with `addon_version`, `gsd_compat.min`, `gsd_compat.max`, and `gsd_compat.tested` fields
- [x] **COMPAT-02**: `install.sh` reads `~/.claude/get-shit-done/VERSION` and compares against `gsd-compat.json` range, printing a warning if outside range
- [x] **COMPAT-03**: Semver comparison function in `install.sh` handles `MAJOR.MINOR.PATCH` format using pure bash arithmetic

### Update Orchestration

- [ ] **UPDT-01**: `bin/gsd-update.sh` runs `npx get-shit-done-cc@latest`, then re-runs `install.sh`, then verifies compatibility
- [ ] **UPDT-02**: Update wrapper records the installed GSD version after update completes
- [ ] **UPDT-03**: Update wrapper exits with clear status: 0=success, 1=GSD update failed, 2=addon reinstall failed, 3=compat warning

## Out of Scope

| Feature | Reason |
|---------|--------|
| Runtime version checking | Install-time only — no overhead during normal use |
| Auto-repair on version mismatch | Warn only — user stays in control |
| CI against every GSD release | GSD publishes too frequently, manual verification sufficient |
| Lock file / hash verification | GSD already provides gsd-file-manifest.json |
| Rollback mechanism | Reinstall is the recovery path |
| Plugin registry | Over-engineering for a single addon |
