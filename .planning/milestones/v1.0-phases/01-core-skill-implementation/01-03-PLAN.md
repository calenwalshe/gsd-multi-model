- After completion, read the generated VERIFICATION.md from the phase directory

**Step 3: Gate on GSD results**
- If GSD verification FAILED:
  - Print the failure details
  - Print: "GSD verification failed. Fix structural issues before cross-review."
  - Print: "Run /gsd:execute-phase to address gaps, then re-run /gsd-codex-verify."
  - Write the partial VERIFICATION.md (GSD results only, cross-review skipped)
  - STOP HERE -- do not proceed to cross-review

**Step 4: Check Codex availability**
- Run `command -v codex` to check if Codex CLI is installed
- If NOT installed:
  - Print warning: "Codex CLI not installed. Skipping cross-model review layer."
  - Print: "GSD verification passed. Cross-review skipped (Codex not available)."
  - Set `CODEX_AVAILABLE=false` -- skip to Step 7 (report)
- If installed: set `CODEX_AVAILABLE=true`

**Step 5: Identify who built what**
- Print: "Step 2/3: Running cross-model review..."
- Check git log for the current phase's commits
- Identify Codex-authored commits (look for "codex" in author/message, or worktree branch merges)
- Identify Claude-authored commits (everything else in the phase)
- This determines what each reviewer focuses on

**Step 6: Run cross-model reviews**

**6a: Claude reviews Codex's autonomous output**
- If Codex-built changes exist:
  - Read those specific changes
  - Verify against PLAN.md and REQUIREMENTS.md specs
  - Check for: incomplete implementations, missing edge cases, test gaps
  - Record findings with CRITICAL/WARNING/INFO severity
- If no Codex-built changes: record "N/A - no Codex-built changes"

**6b: Codex reviews Claude's complex work (via JSONL)**
- Only if `CODEX_AVAILABLE=true`
- Construct review prompt including requirements context, phase context, focus area
- Execute via Bash with JSONL output:
  ```bash
  timeout {timeout_seconds} codex exec --full-auto --json "review prompt..." 2>/tmp/codex-verify-stderr.txt | tee /tmp/codex-verify-output.jsonl
  ```
- Parse the JSONL output line-by-line:
  - For each line, attempt to parse as JSON
  - If parse fails (malformed line): log and skip, do NOT crash
  - Track `has_error=false` and `has_completed=false`
  - If `.type == "error"`: set `has_error=true`, extract `.message`
  - If `.type == "turn.completed"`: set `has_completed=true`, extract `.message` as the review output
  - Ignore other event types (turn.started, item.file_write, etc.)
- After parsing:
  - If `has_error=true`: Codex review = FAIL with error message
  - If `has_completed=true` and no error: parse the review message for CRITICAL/WARNING/INFO findings
  - If neither completed nor error (timeout/truncated): Codex review = INCOMPLETE, note partial output
- Handle timeout (exit 124): "Codex review timed out after {N}m. Results may be incomplete."

**Step 7: Build combined report**
- Determine overall status:
  - All PASS (or N/A): overall = PASS
  - Any WARNING but no CRITICAL: overall = PASS (with warnings)
  - Any CRITICAL: overall = FAIL
  - Codex INCOMPLETE: overall = PASS (with caveat that cross-review was incomplete)
- Format the report using ═══ border format (per locked decision):
  ```
  ═══ DUAL-TOOL VERIFICATION RESULTS ═══

  GSD Verifier (specs check):     {PASS|FAIL}
    {summary of GSD findings}

  Claude reviewed Codex's work:   {PASS|FAIL|N/A}
    {summary of Claude's findings on Codex output}

  Codex reviewed Claude's work:   {PASS|FAIL|INCOMPLETE|SKIPPED}
    {summary of Codex's findings on Claude output}

  Overall:                        {PASS|FAIL}
    {recommendation}
  ═══════════════════════════════════════
  ```

**Step 8: Write VERIFICATION.md AND display inline (per locked decision -- BOTH)**
- Write the full report to `{phase_dir}/{phase}-VERIFICATION.md`
  - Include: timestamp, phase, all three review sections, overall status, raw Codex output in appendix (Claude's discretion on length/relevance)
- Display the report inline (the ═══ format from Step 7)
- Print: "Full report: {path to VERIFICATION.md}"

**Step 9: Recommend next action**
- All PASS: "All verification layers passed. Safe to advance phase."
- Warnings only: "Minor issues found. Review findings and decide whether to fix now or defer."
- Any CRITICAL: "Critical issues found. Fix before advancing. Then re-run /gsd-codex-verify."
- On FAIL: report only -- show what failed, user decides next action (per locked decision: NO auto-generated fix tasks)

**Error handling requirements (per locked decisions):**
- Codex CLI missing: graceful skip, GSD verification still runs

- Truncated JSONL (no turn.completed): report INCOMPLETE
- Missing .planning/ files: report which files missing, skip that context
- /gsd:verify-work fails to run: report the error, note GSD verification could not be completed
