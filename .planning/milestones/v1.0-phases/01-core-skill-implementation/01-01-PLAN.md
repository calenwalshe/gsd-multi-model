**Step 1: Parse arguments and detect environment**
- If `$ARGUMENTS` provided, use as project name; otherwise use current directory basename
- Check `--force` flag in arguments (presence means overwrite everything)
- Detect git: if `.git/` missing, run `git init`
- Track two lists: `created_files` and `skipped_files` for the summary

**Step 2: Stack detection**
- Read `package.json` (Node.js): extract `scripts.build`, `scripts.test`, `scripts.dev` commands
- Read `pyproject.toml` (Python): extract build-system, test framework (pytest, etc.)
- Read `Makefile` (generic): extract common targets (build, test, run)
- Read `go.mod` (Go): extract module name
- Read `Cargo.toml` (Rust): extract package name
- Store detected stack info for AGENTS.md template population
- If nothing detected, use placeholder comments instructing user to fill in

**Step 3: Create AGENTS.md**
- Check if AGENTS.md exists AND --force is not set: skip and add to skipped_files
- Otherwise: create AGENTS.md with Build & Test section populated from stack detection
- Include Architecture, Conventions, and Workflow sections per the existing template
- Add to created_files list

**Step 4: Create CLAUDE.md**
- Same idempotency check as AGENTS.md
- Create with GSD Workflow, Dual-Tool Execution, Dual-Tool Verification, Quality Gates sections
- Reference @AGENTS.md for build commands

**Step 5: Create .claude/rules/ directory and rule files**
- Create directory if missing: `.claude/rules/`
- For EACH rule file (gsd-workflow.md, planning-files.md, test-files.md, security.md):
  - If file exists and no --force: skip, add to skipped_files
  - Otherwise: create with content per existing template, add to created_files
- planning-files.md must include YAML frontmatter with `paths: [".planning/**"]`
- test-files.md must include YAML frontmatter with `paths: ["tests/**", "**/*.test.*", "**/*.spec.*"]`
- security.md must include YAML frontmatter with `paths: ["**/*.env*", "**/auth/**", "**/config/**"]`

**Step 6: Global Codex config (one-time)**
- Check if `~/.codex/config.toml` exists:

  - Same idempotency logic; create global Codex instructions if missing

**Step 7: Global Claude preferences (one-time)**

    - If GSD section present: skip
    - If GSD section missing: APPEND the GSD section (do NOT overwrite the file)
  - If missing: create with GSD Workflow section

**Step 8: Install GSD framework**
- Check if `~/.claude/commands/gsd/` or `~/.claude/get-shit-done/` exists
- If GSD not installed: run `npx get-shit-done-cc@latest`
- If already installed: skip with message

**Step 9: Update .gitignore**
- Read existing .gitignore (or create if missing)
- Check if `.claude/settings.local.json` pattern exists
- If missing: append it
- Do NOT overwrite existing entries

**Step 10: Print summary**

  - "Created:" section listing all created_files

  - "Next steps:" section suggesting `/gsd:new-project`
- End with: "Run /gsd:new-project now to initialize GSD planning? (Recommended for new projects)"

- If npx fails (GSD install): report error with manual install instructions, continue
- If ~/.codex/ or ~/.claude/ parent dirs don't exist: create them
- Never leave partial state: if a file write fails, report which file failed
