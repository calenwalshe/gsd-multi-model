  <files>bin/gate-check.sh, .planning/config.json</files>
  <action>
Create `bin/gate-check.sh` -- the gate orchestrator that runs all enabled checks on staged files and produces structured output.

**Input:** Staged files from `git diff --cached --name-only`. Optionally accepts `--plan-path <path>` for structural tests.

**Config:** Read `.planning/config.json` `gates` section to determine which checks are enabled. Add this section to config.json:
```json
{
  "gates": {
    "enabled": true,
    "lint": {
      "enabled": true,
      "command": "",
      "auto_detect": true
    },
    "architecture": {
      "enabled": true,
      "config_path": ".architecture.json"
    },
    "structural": {
      "enabled": true
    },
    "timeout_seconds": 10,
    "on_timeout": "warn"
  }
}
```

**Three gate types:**

1. **Lint gate:** If `gates.lint.command` is set, run it with staged files substituted for `{files}`. If empty but `auto_detect` is true, check for `.eslintrc*`, `pyproject.toml` (ruff), `Makefile` (with lint target). If no linter found, skip with "skipped" status. Only lint staged files, not the whole project.

2. **Architecture gate:** If `.architecture.json` exists (at `gates.architecture.config_path`), call `bin/validate-architecture.sh` passing the staged files. Collect its JSON output.

3. **Structural gate:** If `--plan-path` provided, parse `<structural_tests>` XML blocks from the plan file. Run each check type:
   - `file-exists`: test -f path
   - `file-contains`: grep -qE pattern path
   - `file-not-contains`: ! grep -qE pattern path
   - `executable`: test -x path
   - `json-valid`: node -e "JSON.parse(require('fs').readFileSync('path','utf8'))"
   - `json-has-key`: node -e to check key existence at dot-path
   If no plan path or no structural_tests block, skip.

**Output format:**
- stdout: JSON `{"passed": bool, "duration_ms": N, "gates": [{"name": "lint|architecture|structural", "passed": bool, "files_checked": N, "message": "...", "violations": [...]}]}`
- stderr: Human-readable summary using the actionable error format from research (=== GATE FAILED === header, per-gate PASS/FAIL/SKIPPED, violation details with file/rule/fix)
- Exit code: 0 if all gates pass (or all skipped), 1 if any gate fails

**Important patterns:**
- Use `$HOME/.claude/get-shit-done` not hardcoded paths
- Follow existing bin/ script conventions: ANSI helpers (ok/warn/err), human output to stderr, machine output to stdout
- Timeout: wrap each gate in a timeout (use `timeout` command on Linux). If timeout, respect `on_timeout` setting (warn = pass with warning, fail = block)
- Filter out `.planning/` files from gate checks -- gates only run on source files
- If `gates.enabled` is false in config, exit 0 immediately with `{"passed": true, "gates": [], "skipped": true}`

<task type="auto">
  <name>Task 2: Create validate-architecture.sh and sample .architecture.json</name>
  <files>bin/validate-architecture.sh, .architecture.json</files>
  <action>
Create `bin/validate-architecture.sh` -- validates staged files against `.architecture.json` module boundary rules.

**Input:** Architecture config path as first argument, then list of file paths to check. Example: `bin/validate-architecture.sh .architecture.json src/foo.js bin/bar.sh`

**Architecture schema (.architecture.json for this project):**
```json
{
  "version": "1.0",
  "modules": {
    "skills/*": {
      "description": "Claude Code skills (Markdown instructions)",
      "can_import": ["bin/*"],
      "cannot_import": ["global/*", ".planning/*"]
    },
    "bin/*": {
      "description": "CLI scripts and tools",
      "can_import": ["bin/*"],
      "cannot_import": ["skills/*"]
    },
    "global/*": {
      "description": "Global config templates",
      "can_import": [],
      "cannot_import": ["skills/*", "bin/*"]
    }
  },
  "rules": [
    {
      "name": "no-circular-skill-deps",
      "description": "Skills must not reference other skills as dependencies",
      "from": "skills/*/",
      "cannot_reach": "skills/*/"
    }
  ]
}
```

**Validation logic:**
1. For each input file, determine which module it belongs to (match against module glob patterns using shell pattern matching)
2. Skip files not matching any module (e.g., root-level files, test files)
3. For matched files, scan for import patterns based on file extension:
   - `.sh`: `source` statements, `. ` (dot-source) statements
   - `.js`/`.ts`/`.cjs`/`.mjs`: `require(` and `import ... from` patterns
   - `.md`: Skip entirely (Markdown references are documentation, not runtime deps per research pitfall 3)
4. For each detected import, resolve it relative to project root and check against `cannot_import` rules
5. Also check `rules[]` entries: if file matches `from` pattern, ensure no import matches `cannot_reach`

**Output:**
- stdout: JSON `{"passed": bool, "files_checked": N, "violations": [{"file": "...", "rule": "...", "message": "...", "fix": "..."}]}`
- Exit code: 0 if no violations, 1 if violations found

**Important:**
- Use regex-based import detection (not AST parsing) per research "Don't Hand-Roll" guidance
- Handle relative paths in imports (resolve `../` patterns relative to file location)
- The `fix` field in violations should suggest where to move the shared logic (e.g., "Move shared logic to bin/ or a shared utility")
