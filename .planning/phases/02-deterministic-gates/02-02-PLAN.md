  <files>bin/test-gate-check.sh</files>
  <action>
Create `bin/test-gate-check.sh` -- integration tests for the gate orchestrator.

**Test fixture pattern:** Create temp directories with minimal git repos. Each test:
1. Creates a temp dir, runs `git init`, sets up config
2. Creates files, stages them with `git add`
3. Runs gate-check.sh
4. Asserts on exit code, stdout JSON, stderr output

**Test cases:**

**GATE-01 (lint blocks commit):**
- `test_lint_pass`: Configure lint command as `echo "ok"`, stage a file, gate passes
- `test_lint_fail`: Configure lint command as `exit 1`, stage a file, gate fails with exit 1
- `test_lint_skip_no_command`: No lint command configured, lint gate shows "skipped"
- `test_lint_only_staged_files`: Ensure only staged files (not all files) are passed to linter

**GATE-03 (structural tests):**
- `test_structural_file_exists_pass`: Plan has `<check type="file-exists" path="foo.sh" />`, file exists, passes
- `test_structural_file_exists_fail`: Plan has check, file missing, fails with violation
- `test_structural_file_contains`: Plan checks for pattern in file, pattern present, passes
- `test_structural_file_not_contains`: Plan checks absence of pattern, pattern absent, passes
- `test_structural_no_plan_path`: No --plan-path flag, structural gate skipped

**GATE-04 (actionable errors):**
- `test_error_format_has_file_rule_fix`: On lint failure, JSON violations include file, rule, message, fix fields
- `test_stderr_human_readable`: On failure, stderr contains "GATE FAILED" header and per-gate status
- `test_pass_output_clean`: On success, stderr shows "GATE PASSED" and stdout has `"passed": true`

**General:**
- `test_gates_disabled`: Set `gates.enabled=false` in config, exit 0 with `"skipped": true`
- `test_planning_files_excluded`: Stage a `.planning/foo.md` file, ensure it's filtered out

**Conventions:** Follow existing test patterns from `bin/test-*.sh` files:
- PASS/FAIL counters
- Cleanup function with trap
- Summary at end with total/pass/fail counts
- Exit 1 if any test fails

<task type="auto">
  <name>Task 2: Create test-validate-architecture.sh covering GATE-02</name>
  <files>bin/test-validate-architecture.sh</files>
  <action>
Create `bin/test-validate-architecture.sh` -- unit tests for the architecture validator.

**Test fixture pattern:** Create temp directories with sample files and architecture configs. Each test:
1. Creates a temp dir with a `.architecture.json` and source files
2. Runs validate-architecture.sh with specific file paths
3. Asserts on exit code and JSON output

**Test cases:**

**Clean passes:**
- `test_no_files`: No files passed, passes with 0 files_checked
- `test_clean_bin_script`: A bin/*.sh file that sources another bin script, passes (bin can import bin)
- `test_unmatched_file_skipped`: A root-level file (e.g., `package.json`) not matching any module, skipped

**Violation detection (GATE-02):**
- `test_skill_imports_skill`: A skills/foo/SKILL.md... skip (MD files are skipped). Instead: A skills/foo/helper.sh that sources skills/bar/util.sh -- should violate "no-circular-skill-deps"
- `test_bin_imports_skill`: A bin/tool.sh that sources skills/init-gsd/something.sh -- violates bin cannot_import skills
- `test_global_imports_bin`: A global/config.sh that sources bin/util.sh -- violates global cannot_import bin

**Violation output format:**
- `test_violation_has_required_fields`: Each violation object has file, rule, message, fix
- `test_multiple_violations`: File with two violations reports both

**Edge cases:**
- `test_missing_architecture_file`: Architecture file doesn't exist, exits with error
- `test_relative_import_resolved`: Import with `../` is resolved correctly before checking

**Conventions:** Same PASS/FAIL pattern as other test scripts.
