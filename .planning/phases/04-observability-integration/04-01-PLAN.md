  <name>Task 1: Create query-telemetry.sh orchestrator and config schema</name>
  <files>bin/query-telemetry.sh, .planning/config.json</files>
  <behavior>
    - No config / observability disabled: exit 0, stdout `{"enabled":false,"endpoints":{},"results":[]}`
    - --endpoint NAME queries only that endpoint
    - --json-only suppresses stderr
    - --project-root PATH overrides project root (for testing)
    - --health mode checks endpoint reachability without full query
    - type: "file" dispatches to tail -n LINES path | grep FILTER
    - type: "http" dispatches to curl with timeout, env var substitution in headers/url
    - type: "docker" dispatches to docker logs --tail N container | grep FILTER
    - type: "journalctl" dispatches to journalctl -u UNIT -n N | grep FILTER
    - ${ENV_VAR} patterns in config strings are replaced with environment variable values at runtime
    - Unreachable endpoints produce warning + empty results, never hard failure
  </behavior>
  <action>
    Create bin/query-telemetry.sh following the exact structure of bin/entropy-sweep.sh:

    1. Shebang, set -euo pipefail, header comment block with usage examples
    2. SCRIPT_DIR/PROJECT_ROOT resolution
    3. ANSI color helpers (copy pattern from entropy-sweep.sh)
    4. Arg parsing loop: --endpoint NAME, --json-only, --project-root PATH, --health
    5. Config loading via node -e: read .planning/config.json, extract observability section
    6. If observability missing or enabled=false, output disabled JSON and exit 0
    7. resolve_env_vars() function: replaces ${VAR_NAME} patterns with env values using bash regex
    8. Endpoint dispatch functions:
       - query_docker(): docker logs --tail N CONTAINER 2>&1 | grep -E FILTER; catch failures gracefully
       - query_http(): curl -s --max-time TIMEOUT -H HEADERS URL; parse response_path via node -e; env var substitution on url and headers
       - query_file(): tail -n LINES PATH | grep -E FILTER; handle missing files
       - query_journalctl(): journalctl -u UNIT -n LINES --no-pager | grep -E FILTER; catch permission issues
    9. Health check mode: for each endpoint, test reachability only (docker info, curl --head, file -e, systemctl is-active)
    10. Main loop: iterate configured endpoints (or single --endpoint), dispatch by type, collect results
    11. Output: JSON on stdout with structure {"enabled":true,"endpoints":{...},"results":[{"name":"...","type":"...","lines":[...],"error":null}],"summary":{"total":N,"success":N,"failed":N}}
    12. Human-readable summary on stderr with color-coded status per endpoint

    Update .planning/config.json: add "observability" key with enabled:false and empty endpoints (opt-in default). Example endpoint structure in a comment block at top of query-telemetry.sh.

    Make script executable: chmod +x bin/query-telemetry.sh
  </action>
