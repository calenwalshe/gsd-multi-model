  <name>Task 1: Build check-doc-consistency.sh and update config.json</name>
  <files>bin/check-doc-consistency.sh, .planning/config.json</files>
  <action>
Create `bin/check-doc-consistency.sh` following the gate-check.sh output pattern (JSON stdout, human stderr, exit codes).

The script checks three automatable conventions from AGENTS.md:

1. **No debug/log statements in production code**: grep for `console.log`, `console.debug`, `console.error`, `echo "DEBUG"` in production files (`.sh`, `.js`, `.cjs`, `.ts` in `bin/` and `skills/`). EXCLUDE test files (`test-*.sh`, `*.test.js`, `*.test.ts`) and `.planning/` directory.

2. **Instruction files under 200 lines**: Run `wc -l` on all `SKILL.md`, `AGENTS.md`, and files in `skills/*/rules/*.md`. Flag any file exceeding 200 lines.

3. **Tests exist for features**: For each `bin/*.sh` that is NOT a test file (not `test-*.sh`), check that a corresponding `bin/test-{name}.sh` exists. Flag missing test files.

Output format:
```json
{
  "passed": true/false,
  "findings": [
    {"check": "debug-statements", "file": "bin/demo.sh", "line": 42, "text": "console.log(...)", "severity": "warning"},
    {"check": "line-count", "file": "skills/init-gsd/SKILL.md", "lines": 250, "limit": 200, "severity": "warning"},
    {"check": "missing-tests", "file": "bin/entropy-sweep.sh", "expected_test": "bin/test-entropy-sweep.sh", "severity": "info"}
  ]
}
```

Arguments:
- `--project-root <path>` (defaults to script's parent directory's parent)
- `--agents-file <path>` (defaults to `AGENTS.md` in project root)

Also update `.planning/config.json` to add the `entropy` section:
```json
{
  "entropy": {
    "enabled": true,
    "schedule": "weekly",
    "checks": {
      "doc_consistency": { "enabled": true },

      "stale_todos": {
        "enabled": true,
        "warn_after_days": 30,
        "critical_after_days": 90
      }
    }
  }
}
```

Use `node -e` to merge the new entropy section into the existing config.json (do not overwrite existing keys).

<task type="auto">
  <name>Task 2: Build entropy-sweep.sh orchestrator</name>
  <files>bin/entropy-sweep.sh</files>
  <action>
Create `bin/entropy-sweep.sh` mirroring the `bin/gate-check.sh` pattern as the single entry point for all entropy checks.

The script:

1. **Reads config** from `.planning/config.json` using the node -e pattern from gate-check.sh. Extract `entropy.enabled`, `entropy.schedule`, and per-check enable flags. Default to all-enabled when config section is absent.

2. **Dispatches to check scripts** based on enable flags:
   - If `doc_consistency` enabled: run `bin/check-doc-consistency.sh`, capture JSON stdout
   - If `architecture` enabled: collect all source files (`.sh`, `.js`, `.cjs`, `.ts`, excluding `.git/`, `node_modules/`, `.planning/`), pass to `bin/validate-architecture.sh .architecture.json [files]`
   - If `stale_todos` enabled: run `bin/check-stale-todos.sh` (will be created by Plan 02), capture JSON stdout. If script doesn't exist yet, skip with a note.

3. **Aggregates results** into a single JSON object to stdout:
```json
{
  "sweep_type": "manual",
  "timestamp": "2026-03-11T10:00:00Z",
  "schedule": "weekly",
  "checks": [
    {"name": "doc-consistency", "passed": true, "findings": []},
    {"name": "architecture", "passed": true, "findings": []},
    {"name": "stale-todos", "passed": false, "findings": [...]}
  ],
  "summary": {"total_findings": N, "checks_run": N, "checks_passed": N}
}
```

4. **Human-readable stderr** output with ANSI colors (same helpers as gate-check.sh: ok/warn/err/skip).

5. **Arguments**:
   - `--check <name>` to run a single check (doc-consistency, architecture, stale-todos)
   - `--json-only` to suppress stderr output
   - No arguments = run all enabled checks

6. **Exit codes**: 0 if all checks pass, 1 if any findings exist, 2 if config error.

For ENTR-02 (architecture sweep): collect all project source files using `find` with the proper exclusions, then pass them to `validate-architecture.sh`. This reuses the existing validator without modification -- entropy-sweep.sh just feeds it a full file list instead of staged-only.

Make executable: `chmod +x bin/entropy-sweep.sh`.
  </action>

  </verify>
  <done>entropy-sweep.sh dispatches to check-doc-consistency.sh and validate-architecture.sh, produces aggregated JSON output, respects config enable flags, defaults work when entropy config section is absent.</done>
</task>
