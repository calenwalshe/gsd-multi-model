Create `skills/gsd-drive/drive-workflow.md` -- the detailed orchestration logic (~250-350 lines). This implements the full decision tree and dispatch mechanism.

**Section 1: Determine Target Phases**

Based on TARGET_MODE from SKILL.md:
- `auto`: Read STATE.md current position. Use `node gsd-tools.cjs roadmap analyze` to find first incomplete phase. Set PHASES=[that phase].
- `single` (--phase N): Check if phase N dependencies are met (all prior phases complete per roadmap). If not, prepend prerequisite phases to PHASES list. Set PHASES=[prerequisites..., N].
- `range` (--to N): Set PHASES=[current_position..N], filtering out already-complete phases.

**Section 2: The Drive Loop**

For each phase in PHASES:
```
1. Print banner: "=== DRIVING PHASE {N}: {name} ==="
2. Determine next action (Section 3)
3. Dispatch action (Section 4)
4. Log result to STATE.md drive log (Section 5)
5. Re-read STATE.md and artifacts
6. If phase complete, continue to next phase
7. If error/pause needed, stop with message
```

**Section 3: Determine Next Action (port from state-machine.ts)**

Read STATE.md via `node gsd-tools.cjs state` and check artifacts in `.planning/phases/{phase_dir}/`:

| Condition | Action |
|-----------|--------|
| No CONTEXT.md | Run discuss-phase |
| CONTEXT.md exists, research enabled, no RESEARCH.md | Run research-phase |
| CONTEXT.md exists, no *-PLAN.md files | Run plan-phase |
| PLAN.md files exist, SUMMARY count < PLAN count | Run execute-phase |
| All SUMMARYs exist, no VERIFICATION.md | Run verify-work |
| VERIFICATION.md exists, no UAT.md | Run verify-work (UAT step) |
| UAT.md with PASS | Run transition |
| UAT.md with FAIL + retry_count < 2 | Run execute-phase --gaps-only, then re-verify |
| UAT.md with FAIL + retry_count >= 2 | STOP with error |

Use `ls` and `grep` on actual files, not in-memory tracking. Always re-read from disk.

For artifact checks, use these commands:
```bash
PHASE_DIR=$(node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" init phase-op ${PHASE} 2>/dev/null | grep phase_dir | ...)
ls "$PHASE_DIR"/*-CONTEXT.md 2>/dev/null
ls "$PHASE_DIR"/*-PLAN.md 2>/dev/null
ls "$PHASE_DIR"/*-SUMMARY.md 2>/dev/null
ls "$PHASE_DIR"/*-UAT.md 2>/dev/null
```

**Section 4: Dispatch Action**

Each action dispatches via Skill() call (NOT Agent tool -- keeps nesting flat per locked decision):

- **discuss**: `Skill(skill="gsd:discuss-phase", args="${PHASE} --auto")` -- auto-answer mode, Claude picks all defaults
- **research**: `Skill(skill="gsd:research-phase", args="${PHASE}")` -- only if config.research enabled
- **plan**: `Skill(skill="gsd:plan-phase", args="${PHASE}")`
- **execute**: `Skill(skill="gsd:execute-phase", args="${PHASE}")`
- **execute-gaps**: `Skill(skill="gsd:execute-phase", args="${PHASE} --gaps-only")`
- **verify**: `Skill(skill="gsd:verify-work", args="${PHASE}")`
- **transition**: `Skill(skill="gsd:transition", args="${PHASE}")`

After each Skill() call completes, re-read STATE.md and artifacts from disk to determine outcome. Never rely on Skill() return values.

**Section 5: Drive Log**

After each action, append to STATE.md a drive log entry. Use this format:

```markdown
## Drive Log

| Timestamp | Phase | Step | Result |
|-----------|-------|------|--------|
```

Use `node gsd-tools.cjs current-timestamp` for timestamp. Append via Read + Write (read STATE.md, append row, write back).

**Section 6: Pause Detection**

Pause ONLY when:
- A checkpoint:human-action task is encountered (deploy, configure API key, etc.)
- An unrecoverable error occurs (2 verification retries exhausted)

NEVER pause for: design decisions, technical choices, verification warnings, ambiguous requirements (auto-decide per locked decision).

When pausing, record resume point:
```bash
node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" state record-session --stopped-at "Phase ${PHASE} - awaiting user action" --resume-file ".planning/phases/${PHASE_DIR}/"
```

**Section 7: Verification Retry Logic**

Track retry count as a local variable (not persistent state). Initialize `VERIFY_RETRIES=0` at start of each phase.

On verification failure (UAT.md with FAIL):
1. Increment VERIFY_RETRIES
2. If VERIFY_RETRIES <= 2: run gap closure (`execute-phase --gaps-only`), then re-run verify-work
3. If VERIFY_RETRIES > 2: STOP with message: "Verification failed after 2 retries for Phase {N}. Manual intervention needed. Run /gsd:verify-work {N} to diagnose."

**Section 8: Phase Completion and Cross-Phase Advance**

After transition completes for a phase:
1. Print banner: "=== PHASE {N} COMPLETE ==="
2. If more phases in PHASES list, continue to next
3. If all targeted phases complete, proceed to final summary

**Anti-patterns to explicitly warn against in the markdown:**
- Do NOT use Agent() for step dispatch -- use Skill() only (avoids #686 nesting freeze)
- Do NOT track state in memory across loop iterations -- always re-read from disk
- Do NOT mention `/clear` to the user
- Do NOT print verbose output between steps -- banners only
